## Why this matters

`@types/node` is pinned at `24.13.2` while the current major is 26.x — two majors behind. The lag matters mainly because this repo uses newer Node API surfaces whose typings evolve, notably `node:sqlite`'s `DatabaseSync` (`src/thread/watermark.ts:2`, `src/sessions/opencode-source.ts:1`); stale types can hide signature changes. It's a type-only dev dependency: `tsc` surfaces any breakage immediately.

## Current state

- `package.json` devDependencies: `"@types/node": "24.13.2"` (exact pin — the repo pins exact versions).
- Runtime Node is 24 (`profiles/base/mise.toml: node = "24"`); `@types/node` 26.x still types the APIs used here — if the bump surfaces types for APIs newer than Node 24 being *used*, that's a real finding (see STOP conditions).
- `pnpm-workspace.yaml` enforces `minimumReleaseAge: 4320` (3 days) — pnpm itself refuses too-fresh releases. Repo/user preference: choose a release older than 7 days.
